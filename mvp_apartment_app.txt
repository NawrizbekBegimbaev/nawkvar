MVP: Приложение аренды квартир

Цель
Сделать простой сервис:
- карта с квартирами
- просмотр информации
- добавление объявления
- редактирование
- статус “продано”

Стек:
- Backend: Django + DRF
- Web: React
- Карта: Yandex Maps API
- Mobile: Flutter (позже)


Бизнес-логика

1. Пользователь открывает приложение
- загружается карта
- отправляется запрос: GET /apartments
- отображаются точки (только ACTIVE)

2. Пользователь нажимает на точку
Открывается popup:
- цена
- фото
- комнаты
- описание
- кнопка “написать” -> окно с двумя кнопками:
  - “Telegram” (ссылка на Telegram владельца)
  - “Позвонить” (звонок на телефон владельца)

3. Кнопка “Я сдаю”
Если НЕ авторизован:
- открывается регистрация
Если авторизован:
- открывается кабинет

4. Регистрация / логин
Минимум:
- телефон
- пароль
Авторизация:
- JWT (access + refresh токены)
- access токен хранится на клиенте (localStorage / secure storage)
- refresh токен для обновления access
После успеха:
- редирект в кабинет

5. Кабинет пользователя
Экран содержит:
- список “Мои объявления”
- кнопка “Добавить объявление”

6. Добавление объявления
Форма:
- цена
- комнаты
- описание
- фото
- локация (через карту)
Действие:
- POST /apartments
Результат:
- объявление появляется на карте

7. Список “Мои объявления”
Запрос:
- GET /my/apartments
Отображение:
- список
- статус (ACTIVE / SOLD)

8. Редактирование объявления
Действие:
- кнопка “Редактировать”
- открывается форма
Запрос:
- PUT /apartments/{id}

9. Отметить “Продано”
Действие:
- кнопка “Продано”
- popup: “точно?”
Если “Да”:
- PATCH /apartments/{id}/status
- status = SOLD
Результат:
- исчезает с карты


Модели данных

User
- id
- phone
- telegram (username, опционально)
- password
- created_at
- updated_at

Apartment
- id
- title
- price
- rooms
- description
- latitude
- longitude
- owner_id
- status (ACTIVE / SOLD)
- created_at
- updated_at

ApartmentImage
- id
- apartment_id
- image_url (хранение: локально в media/)
- updated_at


API

Public
- GET /apartments
- GET /apartments/{id}

Auth
- POST /auth/register
- POST /auth/login

User
- GET /my/apartments

Apartment
- POST /apartments
- PUT /apartments/{id}
- PATCH /apartments/{id}/status
- DELETE /apartments/{id}


Логика карты
- фронт получает список квартир
- отображает точки по lat/lon
- при клике открывает popup


Ограничения MVP
- только аренда
- только ACTIVE объявления на карте
- без платежей
- без чатов
- без отзывов


Базовые правила
- редактировать может только владелец
- без фото объявление нельзя публиковать
- SOLD объявления не отображаются


План запуска
1. Сделать backend (модели + API)
2. Подключить карту
3. Отобразить точки
4. Сделать popup
5. Добавить форму создания
6. Добавить кабинет
7. Заполнить тестовыми данными


Результат MVP
Рабочее приложение, где:
- видно квартиры на карте
- можно открыть и посмотреть
- можно добавить свою
- можно редактировать
- можно отметить “продано”
